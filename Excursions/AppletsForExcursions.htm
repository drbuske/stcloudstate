Suggested Applets for Excursions in Modern Mathematics by Tannenbaum

 

Chapter 1:

 

Applet #1 Suggestion: An applet that will allow a user to input various preference ballots easily and will return

1)      the winner of the election using each of the four methods of chapter 1

2)      a recursive and extended ranking of the candidates using these four methods

Other elements of this applet could include a “random” button for a random election with N candidates.  Having each candidate color-coded might be helpful in showing the various winners using each method.

 

Applet #2 Suggestion:  An applet that will step the user through each step of the four recursive ranking methods mentioned in chapter 1.  Radio buttons would allow users to decide which recursive method to use.  The closest applet I could find was the 2nd part of Example #2 below.  However, I think one needs to be able to modify the original tableau and the comparison is not necessary.  A simple step by step illustration of the recursive ranking methods on both random elections and those elections that a user might enter would be helpful.

This site is nice in that it is directly tied to the textbook since it incorporates all four voting methods discussed in chapter 1.  The applet is certainly usable and contains most of the elements mentioned above.  Still, this applet needs to be “polished.”

 

  1. http://people.uncw.edu/brownj/voting/exc.html

 

This site is nice is that it, too, is directly tied to the textbook since it incorporates all four voting methods discussed in chapter 1.  The interface is easy to understand and open to various examples.  There are 3 applets at this site and all three are useful.

Chapter 2:

 

Applet #1 Suggestion:  An applet that will compute the Banzhaf and Shapley-Shubik indices for any (reasonably sized) weighted voting system.  The example in #1 is excellent; however, it needs to not allow for systems that are not legitimate (i.e. those in which the quota is either too big or too small).  The example in #1 also could be “polished” (e.g. allowing for random systems at the click of a button).  A visual output of the results (with bar or pie charts as in #3) would be nice, but is not essential.

 

Applet #2 Suggestion:  An applet that steps the user through the process of computing the Banzhaf power index for up to 5 players by circling the critical player in each winning coalition.  The applet would allow the user to input the weighted voting system (up to 5 players) and would list all coalitions (up to 31 possible), highlight the winning coalitions, and circle the critical players in each of the winning coalitions.  The applet would then give the final power distribution.  A similar applet guiding the user through the process of computing the Shapley-Shubik power index for up to 4 players would be nice.  It would allow the user to input the weighted voting system (up to 4 players) and would list all sequential coalitions (up to 24 possible) and circle the pivotal player in each.  The applet would then, similarly, give the final power distribution.

 

  1. http://www.cut-the-knot.com/Curriculum/SocialScience/PowerIndex.shtml

 

One downside of this applet is that the number of players is limited to 4.  It would be nice to add one or two additional players.  It also doesn’t show the critical players or the pivotal players involved in each calculation.  The interface is suitable for this level student.

 

  1. http://cow.math.temple.edu/~cow/cgi-bin/manager

 

This applet only handles the Banzhaf power index.  However, if one could strip away much of the garbage in the page and leave just the applet, one would find an easy interface for students.  If an applet were to be written from scratch, this would be an easy one to write I suspect.  A parallel applet for the Shapley-Shibik power index would be nice as well – though it would have to limit the number of players that could be entered.

 

  1. http://www.misojiro.t.u-tokyo.ac.jp/~tomomi/cgi-bin/vpower/index-e.cgi

 

This applet handles the Shapley-Shubik power index (along with other indexes such as Deegan-Packel and what appears to be two versions of the Banzhaf power index).  The presentation is nice in that it gives the user a visual weighting of the power distribution.  This idea could certainly be incorporated into an applet.

Chapter 8

 

Applet #1 Suggestion: An applet that applies the backflow algorithm to find the critical path for a project digraph.  This applet could be similar to Example 1 from chapter 5 in style.  A user would “build” their own project digraph (I like the idea of “sliding” arrows on each weighted edge to change the weight of each edge as in Example #1 below) and then the backflow algorithm could be “stepped” through to find the critical path.   As an alternative, various examples of project digraphs could be given and the user could try to find the critical path by clicking on the appropriate edges (similar to Suggested applet #1 of chapter 7).

 

Applet #2 Suggestion:  An applet that will step through the Decreasing Time Algorithm applied to various projects (such as the MHU project mentioned in the text).  The idea is to illustrate the process shown in Table 8-3 of the text.  For several (well-chosen) project digraphs, the applet builds (one step at a time on the user’s click of a button) a status schedule for two, three, or four processors (as the user chooses).  As each task is added to the schedule, each vertex in the digraph is colored according to the priority list.  That is, colored according to a) ready tasks, b) tasks currently being executed, and c) completed tasks.

Chapter 9

 

Applet #1 Suggestion:  An applet that allows the user to change the first two numbers in the Fibonacci sequence and illustrates the result.  The recursive sequence will have its first two terms determined by the user (using scroll bars) and will output the first 15 terms of the sequence A_N determined by adding the previous two terms at each step.  In a graph below the sequence, the ratios A_N/A_(N-1) would be plotted illustrating the limiting behavior to the golden ratio.  An even more interesting applet would also allow the user to change the format of A_N.  For example, the formula A_N = x * A_(N-1) + y * A_(N-2) + z could be given with x=1, y=1, and z=0 initially.  The user could then be allowed to change these values and observe the behavior of the resulting sequence.

 

Applet #2 Suggestion: A applet consisting of a 15 by 15 grid in which a 3 by 3 square is drawn.  The user would then be asked to highlight “boxes” in such a way as to form a gnomon.  If the user highlighted an appropriate set of these cells, the gnomon would blink showing the user success.  Since there are essentially three gnomonic shapes possible (“L” shapes, “C” shapes, and “O” shapes), the applet could continue until all of these types are found.  An extension of this applet would be to a 3 by 6 square.  In that case, there are four gnomonic shapes possible (a 6 by 9 rectangle is also possible).

 

  1. http://dev1.epsb.edmonton.ab.ca/math14_Jim/math9/strand3/3202.htm

 

This link revolves around similar triangles and as such is not what is needed.  However, similar Java applets are easy to write using Geometer’s Sketchpad.

Chapter 13

 

Applet #1 Suggestion:  An applet illustrating the concept of capture-recapture.  A two-dimensional “urn” is shown containing, say, 200 circles.  The user is allow to “capture” a number of circles (between 5 and 150 say) at random according to their will (use a scroll bar to let the user decide on the size of capture).  The user will then “tag” these objects (turning them a different color).  The user will then “recapture” a number of circles (again, any number between 5 and 150 for example) at random according to their will.   Three different colors will show 1) those captured initially and not recaptured (red), 2) those captured initially and then recaptured (purple), and 3) those not captured initially, but found in the recapture (blue).  Output on the number in each capture, and the number tagged in the recapture would be output.  Also output would be the estimate for the number of “circles” in the urn.  An added suggestion is to let the user repeat their calculations thereby giving a list of estimates from which to draw conclusions.

 

Applet #2 Suggestion:  An applet illustrating the concept of a survey.  This applet can be modeled after Exercises #3 and #4 from chapter 13.  A jar full of gumballs of two (or more depending on user input!) different colors holds a particular number of gumballs (say 200, but this can be decided by the user).  The applet would step through the process of estimating how many gumballs are, let’s say, red.  The user would be allowed to select a certain number (but not all) of the gumballs from the jar and the applet would show a distribution of the colors in graphical and numerical form.  The applet would then show a dynamic calculation determining how many “red” gumballs are in the jar.  Upon a button click, the applet could output how many “red” gumballs were actually in the jar and the sampling error.

Chapter 16

 

Applet #1 Suggestion:  An applet that plots a normal distribution for any (variable) standard deviation s.  The applet would then allow the user to “pull” on the dotted lines of Figure 16-8 from the text to various multiples of s (1s, 1.5s, 1.6s, 2s, etc…) and will shade the area under the normal curve that is within that many standard deviations of the mean.  It will also output the percentage of area shaded under the normal curve as in Figure 16-8.  This applet then would illustrate, dynamically, the 68-95-99.7 rule (and more!). 

 

Applet #2 Suggestion:  An applet that will give the user the output of Example 8 from chapter 16.  That is, an applet that simulates the experiment of tossing of a (fair or dishonest) coin 100 times.  The user would be allowed to change 1) the value of p and 2) the number of times the experiment is repeated (1-10,000 times).
